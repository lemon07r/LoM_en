# Legend of Mortal English Combined

This fork packages a combined English translation mod for **Legend of Mortal** (`活侠传` / `活俠傳`). It uses [`velvetcakes1/LoM_en`](https://github.com/velvetcakes1/LoM_en) as the playable mod base and imports structured translation data from [`joshfreitas1984/LegendOfMortalOverLlm`](https://github.com/joshfreitas1984/LegendOfMortalOverLlm), including the large StringTable revision from PR #1.

The goal is a practical installable English mod with the stable BepInEx/XUnity setup, translated textures, UI fixes, and tested runtime strings from `LoM_en`, plus broader story, legend, event, and StringTable coverage from the LLM translation project.

## Why Use This Fork

- Keeps the existing `LoM_en` mod packaging that is already playable.
- Adds `Mods/English/StringTable.csv`, covering text that XUnity often cannot capture at runtime.
- Uses a generated authoritative XUnity dictionary instead of relying on manual/autogenerated file ordering.
- Keeps translated textures, `UI.resizer.txt`, regex rules, BepInEx, XUnity.AutoTranslator, and loader files from `LoM_en`.
- Includes reproducible merge reports so the source choices and validation results can be audited.

This is still an unofficial fan translation. Large generated translation changes should be treated as playable but not manually perfect.

## Download

Use the latest release from this fork:

<https://github.com/lemon07r/LoM_en/releases>

Download `LegendOfMortal-English-combined.zip`, extract it, and copy the extracted files into the game directory.

## Installation

1. Back up or remove any existing `BepInEx` folder in the game directory if you are updating from another translation mod.
2. Extract `LegendOfMortal-English-combined.zip`.
3. Copy the extracted files into the Legend of Mortal game directory, next to `Mortal.exe`.
4. Start the game.
5. In game, select the Simplified Chinese language option. The mod translates from `zh-CN` to English.

The installed game directory should contain files and folders like:

```text
Legend_of_Mortal/
├── BepInEx/
├── Mods/
│   └── English/
│       └── StringTable.csv
├── .doorstop_version
├── doorstop_config.ini
├── Mortal.exe
└── winhttp.dll
```

## Rebuild

The generated mod files and release zip are built by:

```bash
uv run tools/build_combined_mod.py
```

By default the script expects `LegendOfMortalOverLlm` as a sibling checkout:

```text
translations/
├── LoM_en/
└── LegendOfMortalOverLlm/
```

To use a different import checkout:

```bash
uv run tools/build_combined_mod.py --import-root /path/to/LegendOfMortalOverLlm
```

The build writes:

- `Mod/BepInEx/Translation/en/Text/Translation zh-CN to en.txt`
- `Mod/BepInEx/Translation/en/Text/_AutoGeneratedTranslations.txt`
- `Mod/Mods/English/StringTable.csv`
- `reports/merge_report.md`
- `reports/merge_report.json`
- `dist/LegendOfMortal-English-combined.zip`

`dist/` is ignored and is used only for local release artifacts.

## Merge Policy

The merge script validates candidate translations and rejects malformed entries, empty values, instruction leaks, untranslated Chinese in English values, broken placeholders, broken rich text tags, bad escaped newlines, dangling escapes, YAML fragments, and obvious source-copy output.

It prefers:

- `LoM_en` for short UI, menus, combat/system strings, duplicated runtime strings, and other player-tested text.
- `LegendOfMortalOverLlm` for long story, legend, event, and StringTable text when validation passes.
- glossary-consistent terms from `LegendOfMortalOverLlm/Files/Glossary.yaml` for exact short-term matches.
